# 噪音反击系统

一个 macOS 桌面小工具：选择麦克风、选择反击音频，开始监听后先校准房间底噪，再根据“高于底噪的分贝差、突变强度、低频占比和综合评分”判断噪声冲击。连续达到设定次数后播放反击音频，并进入冷却时间。

软件内置了一个默认提醒音频：`assets/default_alert.wav`。如果不手动选择音频，启动后会自动使用它。

## 运行

```bash
chmod +x run.sh
./run.sh
```

也可以在 Finder 里双击 `启动噪音反击系统.command`。

首次运行 macOS 可能会请求麦克风权限。若没有弹出权限提示，可以到“系统设置 -> 隐私与安全性 -> 麦克风”里允许终端或 Python。

## 参数说明

新版界面分成两栏：左侧是配置，右侧是监控和日志。日常只需要动左侧的设备、音频和基础参数；高级规则用于处理误触发或漏触发。

- `灵敏度`：数值越高越容易触发。
- `监听目标`：勾选要识别的声音类型，可多选。`低频冲击` 适合脚步、震动、撞击；`人声` 会偏向识别说话/喊声；`尖叫声` 会偏向识别高能量高频喊叫。
- `确认次数`：在 4 秒窗口内需要命中的次数，例如 `3次/4秒`。
- `冷却时间`：触发播放后暂停再次触发的秒数。
- `重新校准`：开始监听后，保持环境安静 5 秒，让程序重新学习当前房间底噪。

默认参数明显偏保守：灵敏度 `2`、确认次数 `4次/4秒`、冷却 `12秒`。如果漏触发，再一点点提高灵敏度。

## 高级规则

- `分贝门槛`：当前声音必须比底噪高出多少 dB，越高越不容易触发。
- `低频下限`：低频成分至少占多少，越高越偏向识别脚步、震动、低沉撞击。
- `高频上限`：高频成分最多允许多少，越低越能过滤说话、键盘、尖锐声。
- `连续块数`：连续多少个音频块满足规则才算一次确认，越高越能过滤瞬间误触。

当前识别逻辑是本地实时特征推导，不是 AI 声音分类模型：

- `低频冲击`：看低频能量、低频频谱形状、突变强度，并过滤高频过多的声音。
- `人声`：看人声频段、谐波稳定度、过零率、频谱重心，过滤低频震动和过尖锐噪声。
- `尖叫声`：看尖叫频段、高频能量、频谱重心、突变强度，并过滤低频冲击。

它适合低依赖、本地实时运行；如果要进一步提高人声/尖叫识别准确率，可以再接入机器学习音频分类模型。

## 音频播放

程序会优先用 `soundfile + sounddevice` 播放到选中的输出设备；如果音频格式或设备播放失败，会回退到 macOS 自带的 `afplay` 播放到系统默认输出。
